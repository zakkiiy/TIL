# form_with

### 概要
- フォームを作成するためのヘルパーメソッド。モデルに基づいたフォームを簡単に作成することができる。
- Ajaxを用いた非同期通信にも対応。以下は、@userに紐づいたフォームが作成される。

```ruby
<%= form_with(model: @user, local: true) do |form| %>
  <%= form.label :email %>
  <%= form.text_field :email %>

  <%= form.label :name %>
  <%= form.text_field :name %>

  <%= form.submit %>
<% end %>
```

1. newアクションの場合:
    - コントローラーで `@user = User.new` を実行します。
    - これは空の User インスタンスを生成します。
    - ビューに渡されるこのインスタンスには初期値がないため、フォームのフィールドは空白で表示されます。
    - しかし、このインスタンスが存在することで、form_with はフォームの構造（どのフィールドが必要か、送信先のURLなど）を決定できます。
2. editアクションの場合:
    - コントローラーで `@user = User.find(params[:id])` などを実行します。
    - これは既存のユーザーデータを含んだ User インスタンスを取得します。
    - ビューに渡されるこのインスタンスには既存の値が含まれているため、フォームのフィールドにはそれらの値が表示されます。

form_withの動作:

1. モデルオブジェクト（@user）を受け取ります。
2. オブジェクトが新規レコード（id が nil）か既存レコードかを判断します。
3. 新規レコードの場合は POST リクエスト、既存レコードの場合は PATCH リクエストを生成します。
4. オブジェクトの属性に基づいてフォームフィールドを生成し、存在する値があればそれを表示します。

より詳細な説明:

- newアクション: フォームは空ですが、@userインスタンスの存在により、form_withはUserモデルの構造を理解し、適切なフィールドとアクションを設定できます。送信時には、入力された値が@userの属性にマッピングされてサーバーに送信されます。
- editアクション: @userインスタンスには既存の値が含まれているため、それらの値がフォームに表示されます。ユーザーが値を変更しない限り、既存の値がそのまま維持されます。

この仕組みにより、同じform_withヘルパーを使用して、新規作成と編集の両方のシナリオに対応できるのです。フォームの構造は同じですが、背後にあるインスタンスの状態（新規か既存か）によって、フォームの初期表示と送信後の挙動が適切に変わります。
